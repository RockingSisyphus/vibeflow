# 02. 注册与配置默认值

所有 node 都必须在 registry 中注册。注册时必须提供：

- registry key，也就是 config 中使用的 `type`。
- node class。
- `config_schema`。
- `config_defaults`。

即使 node 没有配置项，也必须显式写空对象。

## 注册示例

```python
from vibeflow import NodeRegistry
from nodes.math_nodes import AddNode, EndNode, SeedNode, StartNode


def build_node_registry() -> NodeRegistry:
    registry = NodeRegistry()
    registry.register("demo.start", StartNode, config_schema={}, config_defaults={})
    registry.register(
        "demo.seed",
        SeedNode,
        config_schema={"value": {"type": "number"}},
        config_defaults={"value": 1},
    )
    registry.register(
        "demo.add",
        AddNode,
        config_schema={"delta": {"type": "number"}},
        config_defaults={"delta": 1},
    )
    registry.register("demo.end", EndNode, config_schema={}, config_defaults={})
    registry.register(
        "demo.no_config",
        SomeNode,
        config_schema={},
        config_defaults={},
    )
    return registry
```

也可以使用装饰器形式，但仍必须提供 `config_schema` 和 `config_defaults`：

```python
registry = NodeRegistry()


@registry.register("demo.add", config_schema={"delta": {"type": "number"}}, config_defaults={"delta": 1})
class AddNode:
    ...
```

长期项目推荐保留显式 `build_node_registry()`，并按 namespace 拆分私有注册函数，例如 `_register_demo_nodes(registry)`。模板 `run.py` 会从 `project/registry.py` 导入并调用 `build_node_registry()`。

## 支持的配置类型

配置 schema 采用轻量 JSON-like 结构。每个字段 schema 至少要有 `type`：

```jsonc
{
  "delta": {"type": "number"},
  "label": {"type": "string"},
  "enabled": {"type": "boolean"},
  "options": {"type": "object"},
  "items": {"type": "array"}
}
```

当前常用类型：

- `any`
- `number`
- `integer`
- `string`
- `boolean`
- `object`
- `array`
- `null`

`bool` 不会被当成 `integer` 或 `number` 接受。

默认值和调用处覆盖值必须是 JSON snapshot safe：可以被 `json.dumps(..., allow_nan=False)` 序列化。不要把模型、tensor、文件句柄、函数或类对象放进 node config；这些对象应作为运行输入或上游 node 输出引用传递。

## 默认值必须完整

`config_defaults` 必须和 `config_schema` 字段完全一致：

```python
config_schema={"delta": {"type": "number"}}
config_defaults={"delta": 1}
```

以下写法会失败：

```python
config_schema={"delta": {"type": "number"}}
config_defaults={}
```

以下写法也会失败：

```python
config_schema={}
config_defaults={"delta": 1}
```

这样做的目的是保证 node 在没有调用处配置时也能运行，并且所有配置项都可审计。

如果 node 没有配置，必须写：

```python
registry.register("demo.start", StartNode, config_schema={}, config_defaults={})
```

## 调用处覆盖配置

```jsonc
{
  "name": "add_small",
  "type": "demo.add",
  "requires": [{"type": "value.in", "cardinality": "exactly_one"}],
  "provides": [{"key": "value.small", "type": "value.out"}],
  "config": {"delta": 1}
}
```

同一 node_type 可以多次调用：

```jsonc
{
  "name": "add_big",
  "type": "demo.add",
  "requires": [{"type": "value.in", "cardinality": "exactly_one"}],
  "provides": [{"key": "value.big", "type": "value.out"}],
  "config": {"delta": 10}
}
```

注意：不同调用点的配置彼此独立。输出 `key` 必须全图唯一；输出 `type` 表达逻辑数据类型，可以重复。下游按 `type` 消费，运行时会在 envelope 中记录实际来源 `key` 和 `source_node`。

## inline 配置简写

内核也支持把非保留字段当作配置覆盖：

```jsonc
{"name": "add", "type": "demo.add", "delta": 3}
```

这等价于：

```jsonc
{"name": "add", "type": "demo.add", "config": {"delta": 3}}
```

为了可读性和减少误判，长期项目推荐使用显式 `config`。保留字段包括 `name`、`type`、`registry_key`、`requires`、`provides`、`config`、`node_configs`、`allow_config_override`、`override_child_config`、`status`、`flow_kind`、`async`、`result_key`。

## 注册错误排查

- `node registration requires config_schema and config_defaults`：注册时漏了 schema 或 defaults。
- `node config schema for 'x' must declare type`：schema 字段缺少 `type`。
- `node config defaults missing keys`：defaults 没覆盖 schema 的所有字段。
- `node config defaults contain undeclared keys`：defaults 出现 schema 未声明字段。
- `node config overrides contain undeclared keys`：config 调用了未在 registry schema 中声明的字段。
- `node config 'x' must match schema type`：默认值或调用处覆盖值类型不匹配。
