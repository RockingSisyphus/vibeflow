# 07. 启动命令与报告

推荐使用模板 `run.py`，因为它会先把 `kernel/vibeflow-kernel.zip` 和 `project/` 加入 `sys.path`，再构建项目自己的 node registry。

不要直接修改 `run.py` 来绕过校验。它在导入内核前会先校验 `kernel/MANIFEST.sha256`，确认内核归档、启动器和 AI 指引没有被改动。

## verify-kernel

```powershell
python run.py verify-kernel
```

只做完整性检查，不加载 config。失败时会列出：

- `Changed`：manifest 中记录的文件内容变了。
- `Missing`：manifest 中记录的文件不存在。
- `Unexpected`：受保护区域出现了 manifest 未记录的文件，例如把内核源码解包到 `kernel/vibeflow/`。

## validate

```powershell
python run.py validate --config project/configs/main.jsonc
```

输出健康报告。状态可能是：

- `PASS`：通过。
- `CONCERNS`：有 warning，但可运行。
- `FAIL`：配置、拓扑或健康检查失败，不应运行。
- `ERROR`：读取、语法、插件加载等错误。

## run

```powershell
python run.py run --config project/configs/main.jsonc --run-root runs
python run.py run --config project/configs/main.jsonc --input input.json --run-root runs
```

运行前会强制健康检查。失败时拒绝运行。

成功或失败都会在运行目录生成报告：

```text
runs/<run_id>/
  input_summary.json
  effective_policy.json
  compiled_graph.json
  health_report.json
  graph.mmd
  graph.txt
  graph.svg
  graph.svg.error.txt      # 仅 SVG 渲染失败时出现
  graph.expanded.svg
  graph.expanded.svg.error.txt  # 仅详细审查 SVG 渲染失败时出现
  runtime_trace.jsonl
  output_summary.json
```

`input_summary.json`、`output_summary.json` 和 `runtime_trace.jsonl` 只写 key、类型、大小、事件和失败摘要，不保存真实 Python 对象内容。训练类对象可以在 node 间按引用传递，但不会被报告序列化。

异步 node 会在 trace 中记录 `async_result` / `async_result_join` 或 `async_detached` / `async_detached_done` / `async_detached_failed` 事件。

`--input` 指向一个 JSON 文件，内容会作为初始 context。适合传入字符串、数字、列表、对象等 JSON 数据。若需要传递模型、tensor、数据库连接等真实 Python 对象，应写自定义启动器调用 `run_checked(..., initial=...)`，不要把这些对象写进 JSON。

运行选项：

```powershell
python run.py run --config project/configs/main.jsonc --runtime-profile debug
python run.py run --config project/configs/main.jsonc --runtime-profile train
python run.py run --config project/configs/main.jsonc --trace boundary --execution compiled --no-node-hooks
python run.py run --config project/configs/main.jsonc --allow-planned-stub
```

- `--runtime-profile debug`：完整 trace，所有 hook 开启，执行模式为 `plan`。
- `--runtime-profile train`：边界 trace，run/block hooks 开启，node/nodeset hooks 关闭，执行模式为 `compiled`。
- `--trace full|boundary|off`：控制 trace 详细程度。
- `--execution plan|block|compiled`：控制执行模式；`compiled` 不适合的图会在内部回退或报错，调试时先用 `plan`。
- `--async-flush-timeout`：run 结束时等待 detached async task 的秒数。
- `--allow-planned-stub`：仅开发测试用。开启后只有 `planned_behavior.kind == "python_stub"` 的 planned node/nodeset 可执行；`blocking` 和 `transparent` 仍拒绝运行。

## mermaid / ascii / svg

```powershell
python run.py mermaid --config project/configs/main.jsonc --output reports/graph.mmd
python run.py mermaid --config project/configs/main.jsonc --expand-nodesets --output reports/graph_full.mmd

python run.py ascii --config project/configs/main.jsonc --output reports/graph.txt
python run.py ascii --config project/configs/main.jsonc --expand-nodesets --output reports/graph_full.txt

python run.py svg --config project/configs/main.jsonc --output reports/graph.svg
python run.py svg --config project/configs/main.jsonc --expand-nodesets --output reports/graph_full.svg
python run.py svg --config project/configs/main.jsonc --expand-nodesets --mermaid-max-text-size 800000 --output reports/graph_full.svg
python run.py svg --config project/configs/main.jsonc --expand-nodesets --review-fragment-max-width 6400 --output reports/graph_full.svg
```

- Mermaid 图用于审查源 flowchart。
- ASCII 图不依赖外部渲染器，适合终端快速查看。
- SVG 图使用 Mermaid CLI 渲染，最适合人工审查标准流程图形状。首次使用前需要在 `tools/mermaid-renderer/` 执行 `npm install`。
- SVG 渲染默认传入 Mermaid 配置：普通图 `maxTextSize=200000`、`maxEdges=2000`；`--expand-nodesets` 图 `maxTextSize=500000`、`maxEdges=5000`，并固定采用 `review-columns` SVG composer，把主流程、plugins、base_lib 和展开 nodeset 分列展示。如果展开图仍超过 Mermaid 限制，可用 `--mermaid-max-text-size` 和 `--mermaid-max-edges` 覆盖；如果希望审查图单个片段显示得更宽，可用 `--review-fragment-max-width` 调整默认 `3200px` 上限。

图形输出会显示：

- 标准 flow_kind 形状。
- node / nodeset 契约 key。
- 中文说明和元数据。
- `when` 条件。
- planned / external 标记。
- health finding 标记。

## inspect-node

```powershell
python run.py inspect-node --type demo.add --module project/nodes/math_nodes.py --class AddNode
```

用于单独检查某个 node class 的纯度、契约和源码规模。

## inspect-config

```powershell
python run.py inspect-config --config project/configs/main.jsonc
```

用于查看解析后的 nodes、edges、nodesets、`max_steps`。

## quality

```powershell
python run.py quality --path project
```

这是通用代码质量检查工具，可检查不基于内核的普通 Python 项目。它会检查文件长度、函数复杂度、依赖链长度、重复 AST 形状等。也可以用它自检使用内核开发的项目。

## 自定义启动器

如果模板 `run.py` 不够用，可以在项目根目录写自己的启动器，但仍应复用相同入口：

```python
from pathlib import Path
import sys

ROOT = Path(__file__).resolve().parent
sys.path.insert(0, str(ROOT / "kernel" / "vibeflow-kernel.zip"))
sys.path.insert(0, str(ROOT / "project"))

from registry import build_node_registry
from vibeflow import RuntimeOptions, run_checked

result = run_checked(
    "project/configs/main.jsonc",
    registry=build_node_registry(),
    initial={"value.in": 1},
    run_root="runs",
    runtime_options=RuntimeOptions(trace="boundary", execution="plan"),
)
print(result.health.status, result.run_dir)
```

自定义启动器也应保留 kernel manifest 校验逻辑；除非你非常清楚后果，不要跳过模板 `run.py` 的完整性检查。
