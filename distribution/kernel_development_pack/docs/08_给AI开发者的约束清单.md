# 08. 给 AI 开发者的约束清单

当你作为 AI 使用这个内核开发程序时，必须遵守下面的清单。

## 开发 node 前

- 先确认这个功能能否拆成一个小纯函数 node。
- 如果需要文件、网络、数据库、浏览器或外部系统，改为设计 boundary，不要写进 node。
- 如果逻辑可复用，放到 `base_lib/` 的纯函数里。
- 如果多个 node 形成模块，写 nodeset，不要让 node 互相调用。

## 写 node 时

- 必须写完整 `NODE_INFO`。
- 必须写完整 `CONTRACT`。
- 必须使用 `run_pure(self, inputs, params)`。
- 只读取 `inputs` 和 `params`。
- 只返回 `CONTRACT.provides` 声明的 key。
- 不修改 `inputs`。
- 不使用动态 output key。
- 不写 public helper 方法。
- 不在 `__init__` 中接收资源或保存外部对象。
- 不导入其他 node。

## 注册 node 时

- 必须提供 `config_schema`。
- 必须提供 `config_defaults`。
- 没有配置也写 `{}`。
- 默认值必须覆盖所有 schema 字段。

## 写 config 时

- 每个调用点都要有唯一 `name`。
- 同一个 node_type 多次调用时，用不同 `name` 和不同 `config`。
- `requires` / `provides` 要和 node 契约一致。
- 环路必须写 `loops` 和 `max_iterations`。
- nodeset 内部配置覆盖使用 `node_configs`。
- 嵌套 nodeset 使用 dotted path，例如 `outer.inner_node`。

## 出错时

优先根据健康报告修正根因：

- schema 错误：修 config 结构。
- topology 错误：修 requires/provides、edge、loop、nodeset。
- purity 错误：拆 node 或迁移副作用到 boundary。
- config 错误：修注册默认值或调用处 `config`。
- base_lib 错误：拆纯 helper，删除副作用和过深依赖。

不要通过扩大 node、加补丁或放宽 policy 来绕过硬规则。
